GF1靶机工具使用说明
一、环境准备
1. 安装 Python
下载 Python 3.6 或更高版本：https://www.python.org/downloads/

安装时务必勾选“Add Python to PATH”（将 Python 添加到系统环境变量），否则后续可能无法运行。

2. 安装依赖库
双击运行 run.bat（同目录下），脚本会自动检测并安装 requests 和 frida 库。

若手动安装：打开命令提示符，输入 pip install requests frida 后，回车。

二、抓取 Token（登录凭证）
启动游戏，进入游戏界面【确保游戏进程名称为 GrilsFrontLine.exe（若为Epic则为 GirlsFrontLine.exe ）】。

双击运行  监视器  文件夹中的 run.py。

程序将自动附加到游戏进程，游戏内执行任意操作后会在窗口中显示捕获到的Token。

将 Token 复制到剪贴板。

★★★★★注意：Token 每次登录游戏都会变化，重新启动游戏后需重新抓取。

三、使用主程序 靶机抓取工具.py
1. 启动工具
双击  靶机抓取工具.py  启动图形界面。若没有 Python 文件关联，可在命令行执行 python 靶机抓取工具.py。

2. 界面说明

服务器地址：下拉选择对应服务器URL地址（M4、15、SOP、RO、M16）。

删除接口地址：自动生成，无需修改。

认证信息：填入 游戏UID 与从监视器工具中得到的 Token。

目标敌人ID列表：通过“添加敌人”输入数字 ID，可逐个添加、编辑、删除、清空。

靶场ID设置：默认不勾选，使用自动顺序 1,2,3...；若勾选可自定义靶场 ID（需与敌人数量匹配）。

日志区域：显示请求发送和结果。

控制按钮：保存/加载配置（不保存 Token）、清空日志。

3. 批量注入靶机
在“目标敌人ID列表”中添加需要抓取的敌人 ID（最多 35 组）。

确认服务器和 Token 已填写。

点击“靶机，启动”按钮，程序将按顺序发送添加请求。日志区会显示每个请求的结果（SUCCESS 或 FAIL）。

★★★★★重新登录游戏后相关靶机数据将同步至游戏内。

★★★★★可以在相应目录保存配置后，在  配置名.json 文件中修改 enemy 数组中的 敌方ID 后再 加载配置 并 启动 以替换需要测试的怪组。

4. 一键清空靶机

点击“一键清空游戏内当前列表中的靶机”，确认后程序发送删除请求。日志会显示删除结果。

★★★★★重新登录游戏后游戏内相关靶机数据将删除。由于傻逼ym的屎山代码，清空靶机后需要大退，否则会卡在查看靶机界面。

如果游戏内没有对应 ID 的靶机，删除请求也会返回空，但日志仍显示 FAIL（因服务器未返回 "1"）。实际不影响。

四、常见问题
Q1: 抓包时提示“Failed to attach”，或找不到游戏进程？
确保游戏已启动且进程名为 GrilsFrontLine.exe（注意大小写）傻逼羽中Steam用的是  GrilsFrontLine.exe  而Epic中用的  GirlsFrontLine.exe，咱也不知道为什么。

以管理员身份运行 run.py 或使用管理员命令行。

Q2: 安装 frida 时提示缺少 Visual C++ 构建工具？
下载并安装 Visual C++ 生成工具：https://visualstudio.microsoft.com/visual-cpp-build-tools/

或从 https://pypi.org/project/frida/#files 下载对应 Python 版本的 wheel 文件手动安装（pip install 文件名.whl）。

Q3: 删除靶机返回空响应（FAIL）？

确认 Token 有效且 UID 正确。

如果游戏内没有对应 ID 的靶机，删除请求也会返回空，但日志仍显示 FAIL（因服务器未返回 "1"）。实际不影响。


五、文件说明

靶机抓取工具.py：主程序（注入靶机 / 清空靶机）。

run.py：抓包工具，获取 Token。

hook_dual.js：Frida 脚本（被 run.py 调用）。

run.bat：依赖安装脚本（安装 requests 和 frida）。

traffic_dumps：抓包时保存的临时 JSON 文件。
